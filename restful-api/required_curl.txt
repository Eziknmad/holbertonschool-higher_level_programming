Consuming Data from an API Using curl
Here's a complete walkthrough of each instruction with expected outputs.

1. Installing and Verifying curl
Install curl (if not already installed):
bash# Ubuntu/Debian
sudo apt-get install curl

# macOS (usually pre-installed, or via Homebrew)
brew install curl

# Check version
curl --version

bashcurl -I https://jsonplaceholder.typicode.com/posts

Make a POST Request (-X POST and -d flags)
bashcurl -X POST \
  -d "title=foo&body=bar&userId=1" \
  https://jsonplaceholder.typicode.com/posts
Expected output:
json{
  "title": "foo",
  "body": "bar",
  "userId": "1",
  "id": 101
}
POST with JSON body and Content-Type header (more realistic/production-style):
bashcurl -X POST \
  -H "Content-Type: application/json" \
  -d '{"title": "foo", "body": "bar", "userId": 1}' \
  https://jsonplaceholder.typicode.com/posts
Expected output:
json{
  "title": "foo",
  "body": "bar",
  "userId": 1,
  "id": 101
}
Other Useful curl Flags
bash# Verbose output - shows full request and response details including SSL handshake
curl -v https://jsonplaceholder.typicode.com/posts/1

# Follow redirects automatically
curl -L http://jsonplaceholder.typicode.com/posts/1

# Save output to a file
curl -o posts.json https://jsonplaceholder.typicode.com/posts

# Set a custom request header (e.g., for auth tokens)
curl -H "Authorization: Bearer YOUR_TOKEN" https://jsonplaceholder.typicode.com/posts

# Make a PUT request (full update)
curl -X PUT \
  -H "Content-Type: application/json" \
  -d '{"id": 1, "title": "updated title", "body": "updated body", "userId": 1}' \
  https://jsonplaceholder.typicode.com/posts/1

# Make a PATCH request (partial update)
curl -X PATCH \
  -H "Content-Type: application/json" \
  -d '{"title": "patched title"}' \
  https://jsonplaceholder.typicode.com/posts/1

# Make a DELETE request
curl -X DELETE https://jsonplaceholder.typicode.com/posts/1
